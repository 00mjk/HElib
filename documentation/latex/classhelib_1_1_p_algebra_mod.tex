\doxysubsection{Detailed Description}
The structure of Z\mbox{[}X\mbox{]}/(Phi\+\_\+m(\+X), p) 

An object of type \mbox{\hyperlink{classhelib_1_1_p_algebra_mod}{P\+Algebra\+Mod}} stores information about a \mbox{\hyperlink{classhelib_1_1_p_algebra}{P\+Algebra}} object z\+M\+Star, and an integer r. It also provides support for encoding and decoding plaintext slots.

the \mbox{\hyperlink{classhelib_1_1_p_algebra}{P\+Algebra}} object z\+M\+Star defines (Z/mZ)$^\wedge$$\ast$ /(0), and the \mbox{\hyperlink{classhelib_1_1_p_algebra_mod}{P\+Algebra\+Mod}} object stores various tables related to the polynomial ring Z/(p$^\wedge$r)\mbox{[}X\mbox{]}. To do this most efficiently, if p == 2 and r == 1, then these polynomials are represented as G\+F2X\textquotesingle{}s, and otherwise as zz\+\_\+pX\textquotesingle{}s. Thus, the types of these objects are not determined until run time. As such, we need to use a class hierarchy, as follows.

\begin{DoxyItemize}
\item \mbox{\hyperlink{classhelib_1_1_p_algebra_mod_base}{P\+Algebra\+Mod\+Base}} is a virtual class\end{DoxyItemize}
\begin{DoxyItemize}
\item P\+Algebra\+Mod\+Derived$<$type$>$ is a derived template class, where type is either P\+A\+\_\+\+G\+F2 or P\+A\+\_\+zz\+\_\+p.\end{DoxyItemize}
\begin{DoxyItemize}
\item The class \mbox{\hyperlink{classhelib_1_1_p_algebra_mod}{P\+Algebra\+Mod}} is a simple wrapper around a smart pointer to a \mbox{\hyperlink{classhelib_1_1_p_algebra_mod_base}{P\+Algebra\+Mod\+Base}} object\+: copying a \mbox{\hyperlink{classhelib_1_1_p_algebra}{P\+Algebra}} object results is a \char`\"{}deep copy\char`\"{} of the underlying object of the derived class. It provides d\+Direct access to the virtual methods of \mbox{\hyperlink{classhelib_1_1_p_algebra_mod_base}{P\+Algebra\+Mod\+Base}}, along with a \char`\"{}downcast\char`\"{} operator to get a reference to the object as a derived type, and also == and != operators. \end{DoxyItemize}
