\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Scheme$>$\newline
class helib\+::\+Ptxt$<$ Scheme $>$}

An object that mimics the functionality of the {\ttfamily \mbox{\hyperlink{classhelib_1_1_ctxt}{Ctxt}}} object, and acts as a convenient entry point for inputting/encoding data which is to be encrypted. 

{\ttfamily \mbox{\hyperlink{classhelib_1_1_ptxt}{Ptxt}}} is templated on {\ttfamily Scheme}, which may be {\ttfamily \mbox{\hyperlink{structhelib_1_1_c_k_k_s}{C\+K\+KS}}} or {\ttfamily \mbox{\hyperlink{structhelib_1_1_b_g_v}{B\+GV}}}.

In the \mbox{\hyperlink{structhelib_1_1_b_g_v}{B\+GV}} case, {\ttfamily \mbox{\hyperlink{classhelib_1_1_ptxt}{Ptxt}}} can be considered to be an element of $\mathbb{Z}_p[x]/\Phi(m)$ viewed as a vector of slots with values each in $\mathbb{Z}_p[x]/G$ where G is one of the irreducible factors of $\Phi(m)$, and all operations are performed entry-\/wise.

General usage\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classhelib_1_1_ptxt}{helib::Ptxt<BGV>}} p1(bgv\_context, data);}
\DoxyCodeLine{\mbox{\hyperlink{classhelib_1_1_ptxt}{helib::Ptxt<BGV>}} p2(bgv\_context, data);}
\DoxyCodeLine{p1 += p2;}
\DoxyCodeLine{std::cout << p1 << std::endl;}
\end{DoxyCode}


Internally, {\ttfamily \mbox{\hyperlink{classhelib_1_1_ptxt}{Ptxt}}} objects store their data as a {\ttfamily std\+::vector$<$\mbox{\hyperlink{classhelib_1_1_poly_mod}{helib\+::\+Poly\+Mod}}$>$}, where {\ttfamily \mbox{\hyperlink{classhelib_1_1_poly_mod}{Poly\+Mod}}} is a convenience type representing an element of the above ring, $\mathbb{Z}_p[x]/G$. The {\ttfamily \mbox{\hyperlink{classhelib_1_1_poly_mod}{Poly\+Mod}}} type can be easily converted via {\ttfamily static\+\_\+cast} to more convenient types such as {\ttfamily long} and {\ttfamily N\+T\+L\+::\+Z\+ZX}.

In the {\ttfamily \mbox{\hyperlink{structhelib_1_1_c_k_k_s}{C\+K\+KS}}} case, the slot type is {\ttfamily std\+::complex$<$double$>$}, and has sensible operator overloads supporting operations with other {\ttfamily \mbox{\hyperlink{classhelib_1_1_ptxt}{Ptxt}}$<$\mbox{\hyperlink{structhelib_1_1_c_k_k_s}{C\+K\+KS}}$>$}, {\ttfamily \mbox{\hyperlink{classhelib_1_1_ctxt}{Ctxt}}}, and {\ttfamily std\+::complex$<$double$>$} objects, as well as performing all operations slot-\/wise.

A large number of operator overloads are defined so that {\ttfamily \mbox{\hyperlink{classhelib_1_1_ptxt}{Ptxt}}} objects should easily inter-\/operate, as well as providing interoperability with other logically compatible types e.\+g. {\ttfamily long} and {\ttfamily N\+T\+L\+::\+Z\+ZX} in the \mbox{\hyperlink{structhelib_1_1_b_g_v}{B\+GV}} case, {\ttfamily std\+::complex$<$double$>$} in the {\ttfamily \mbox{\hyperlink{structhelib_1_1_c_k_k_s}{C\+K\+KS}}} case, and {\ttfamily \mbox{\hyperlink{classhelib_1_1_ctxt}{helib\+::\+Ctxt}}} in both cases.

For example if we have a plaintext {\ttfamily ptxt} such that {\ttfamily ptxt\mbox{[}0\mbox{]}=slot0}, {\ttfamily ptxt\mbox{[}1\mbox{]}=slot1}, {\ttfamily ptxt\mbox{[}2\mbox{]}=slot2}, and {\ttfamily ptxt\mbox{[}i\mbox{]}=0} for {\ttfamily i$>$2}, it will be serialized as \textquotesingle{}\mbox{[}\textquotesingle{}slot0\textquotesingle{}, \textquotesingle{}slot1\textquotesingle{}, \textquotesingle{}slot2\textquotesingle{}, {\ttfamily 0}, {\ttfamily 0} ...\mbox{]}\textquotesingle{}.
