\doxysubsection{Detailed Description}
A class representing the logic of the order of bit products when adding two integers. 

Given two input arrays a\mbox{[}\mbox{]}, b\mbox{[}\mbox{]}, we build a D\+AG with each node representing a term either of the form p\+\_\+\{i,j\} = \textbackslash{}prod\+\_\+\{t=j\}$^\wedge$i (a\mbox{[}t\mbox{]}+b\mbox{[}t\mbox{]}), or of the form q\+\_\+\{i,j\} = (a\mbox{[}j\mbox{]}$\ast$b\mbox{[}j\mbox{]}) $\ast$ \textbackslash{}prod\+\_\+\{t=j+1\}$^\wedge$i (a\mbox{[}t\mbox{]}+b\mbox{[}t\mbox{]}). The source nodes are of the forms (a\mbox{[}i\mbox{]}$\ast$b\mbox{[}i\mbox{]}) and (a\mbox{[}i\mbox{]}+b\mbox{[}i\mbox{]}), and each non-\/source node has exactly two parents, whose product yields that node.

When building the D\+AG, we keep the level of each node as high as possible. For example we can set q\+\_\+\{i,j\}=p\+\_\+\{i,k\}$\ast$q\+\_\+\{k-\/1,j\} or q\+\_\+\{i,j\}=p\+\_\+\{i,k+1\}$\ast$q\+\_\+\{k,j\} (among other options), and we choose the option that results in the highest level. In addition, we try to minimize the number of nodes in the D\+AG that actually need to be computed while adding the two numbers (subject to still consuming as few levels as possible).
