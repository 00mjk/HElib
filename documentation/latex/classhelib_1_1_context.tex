The different columns in any key-\/switching matrix contain encryptions of multiplies of the secret key, sk, B1$\ast$sk, B2$\ast$\+B1$\ast$sk, B3$\ast$\+B2$\ast$\+B1$\ast$sk,... with each Bi a product of a few \char`\"{}non-\/special\char`\"{} primes in the chain. The digits data member indicate which primes correspond to each of the Bi\textquotesingle{}s. These are all \mbox{\hyperlink{classhelib_1_1_index_set}{Index\+Set}} objects, whose union is the subset ctxt\+Primes.

The number of Bi\textquotesingle{}s is one less than the number of columns in the key switching matrices (since the 1st column encrypts sk, without any Bi\textquotesingle{}s), but we keep in the digits std\+::vector also an entry for the primes that do not participate in any Bi (so digits.\+size() is the same as the number of columns in the key switching matrices).
