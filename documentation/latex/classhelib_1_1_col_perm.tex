\doxysubsection{Detailed Description}
Permuting a single dimension (column) of a hypercube. 

\mbox{\hyperlink{classhelib_1_1_col_perm}{Col\+Perm}} is derived from a \mbox{\hyperlink{classhelib_1_1_hyper_cube}{Hyper\+Cube$<$long$>$}}, and it uses the cube object to store the actual permutation data. The interpretation of this data, however, depends on the data member dim.

The cube is partitioned into columns of size n = get\+Dim(dim)\+: a single column consists of the n entries whose indices i have the same coordinates in all dimensions other than dim. The entries in any such column form a permutation on \mbox{[}0..n).

For a given \mbox{\hyperlink{classhelib_1_1_col_perm}{Col\+Perm}} perm, one way to access each column is as follows\+: for slice\+\_\+index = \mbox{[}0..perm.\+get\+Prod(0, dim)) \mbox{\hyperlink{classhelib_1_1_cube_slice}{Cube\+Slice}} slice(perm, slice\+\_\+index, dim) for col\+\_\+index = \mbox{[}0..perm.\+get\+Prod(dim+1)) get\+Hyper\+Column(column, slice, col\+\_\+index)

Another way is to use the get\+Coord and add\+Coord methods.

For example, permuting a 2x3x2 cube along dim=1 (the 2nd dimension), we could have the data std\+::vector as \mbox{[} 1 1 0 2 2 0 2 0 1 1 0 2 \mbox{]}. This means the four columns are permuted by the permutations \mbox{[} 1 0 2 \mbox{]} \mbox{[} 1 2 0 \mbox{]} \mbox{[} 2 1 0 \mbox{]} \mbox{[} 0 1 2 \mbox{]}. Written explicitly, we get\+: \mbox{[} 2 3 0 5 4 1 10 7 8 9 6 11 \mbox{]}.

Another representation that we provide is by \char`\"{}shift amount\char`\"{}\+: how many slots each element needs to move inside its small permutation. For the example above, this will be\+: \mbox{[} 1 -\/1 0 \mbox{]} \mbox{[} 2 -\/1 -\/1 \mbox{]} \mbox{[} 2 0 -\/2 \mbox{]} \mbox{[} 0 0 0 \mbox{]} so we write the permutation as \mbox{[} 1 1 -\/1 1 0 -\/2 2 0 0 0 -\/2 0 \mbox{]}.
