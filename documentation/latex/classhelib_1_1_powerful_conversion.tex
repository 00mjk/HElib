\doxysubsection{Detailed Description}
Conversion between powerful representation in R\+\_\+m/(q) and zz\+\_\+pX. 

Usage pattern is as follows\+:

// compute tables for index translation \mbox{\hyperlink{classhelib_1_1_powerful_translation_indexes}{Powerful\+Translation\+Indexes}} ind(mvec); // mvec is some factorization of m

// ... set the current zz\+\_\+p\+::modulus to something before initializing \mbox{\hyperlink{classhelib_1_1_powerful_conversion}{Powerful\+Conversion}} p\+Conv(ind);

// Alternatively use // \mbox{\hyperlink{classhelib_1_1_powerful_conversion}{Powerful\+Conversion}} p\+Conv(); p\+Conv.\+init\+P\+Conv(ind); // Only the latter call needs zz\+\_\+p\+::modulus to be defined

// A powerful basis is defined wrt same modulus and cube signature Hyper\+Cube$<$zz\+\_\+p$>$ powerful(p\+Conv.\+get\+Short\+Sig());

// ... some code here to initialize powerful // code can also do other stuff, perhaps changing zz\+\_\+p\+::modulus

p\+Conv.\+restore\+Modulus(); // restore zz\+\_\+p\+::modulus zz\+\_\+pX poly; // defined relative to the same modulus p\+Conv.\+powerful\+To\+Poly(poly, powerful);

// ... some more code here, perhaps changing zz\+\_\+p\+::modulus again

p\+Conv.\+restore\+Modulus(); // restore zz\+\_\+p\+::modulus p\+Conv.\+poly\+To\+Powerful(powerful, poly);
