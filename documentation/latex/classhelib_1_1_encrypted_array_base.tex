\doxysubsection{Detailed Description}
virtual class for data-\/movement operations on arrays of slots 

An object ea of type \mbox{\hyperlink{classhelib_1_1_encrypted_array}{Encrypted\+Array}} stores information about an \mbox{\hyperlink{classhelib_1_1_context}{Context}} context, and a monic polynomial G. If context defines parameters m, p, and r, then ea is a helper abject that supports encoding/decoding and encryption/decryption of std\+::vectors of plaintext slots over the ring (Z/(p$^\wedge$r)\mbox{[}X\mbox{]})/(G).

The polynomial G should be irreducible over Z/(p$^\wedge$r) (this is not checked). The degree of G should divide the multiplicative order of p modulo m (this is checked). Currently, the following restriction is imposed\+:

either r == 1 or deg(\+G) == 1 or G == factors\mbox{[}0\mbox{]}.

ea stores objects in the polynomial ring Z/(p$^\wedge$r)\mbox{[}X\mbox{]}.

Just as for the class \mbox{\hyperlink{classhelib_1_1_p_algebra_mod}{P\+Algebra\+Mod}}, if p == 2 and r == 1, then these polynomials are represented as G\+F2X\textquotesingle{}s, and otherwise as zz\+\_\+pX\textquotesingle{}s. Thus, the types of these objects are not determined until run time. As such, we need to use a class hierarchy, which mirrors that of \mbox{\hyperlink{classhelib_1_1_p_algebra_mod}{P\+Algebra\+Mod}}, as follows.

\mbox{\hyperlink{classhelib_1_1_encrypted_array_base}{Encrypted\+Array\+Base}} is a virtual class

Encrypted\+Array\+Derived$<$type$>$ is a derived template class, where type is either P\+A\+\_\+\+G\+F2 or P\+A\+\_\+zz\+\_\+p.

The class \mbox{\hyperlink{classhelib_1_1_encrypted_array}{Encrypted\+Array}} is a simple wrapper around a smart pointer to an \mbox{\hyperlink{classhelib_1_1_encrypted_array_base}{Encrypted\+Array\+Base}} object\+: copying an \mbox{\hyperlink{classhelib_1_1_encrypted_array}{Encrypted\+Array}} object results is a \char`\"{}deep copy\char`\"{} of the underlying object of the derived class.
